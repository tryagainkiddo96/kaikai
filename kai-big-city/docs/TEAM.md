# Kai Big City Team

This is the active dev team model for this project.

## Core Team

- Creative Director (`You`)
  - Owns vision, tone, lore, character authenticity.
  - Final call on priorities and feel.
- Technical Director (`Codex`)
  - Owns implementation, systems quality, integration, and shipping loops.
  - Breaks features into milestones and executes continuously.
- Gameplay Designer (`Codex`)
  - Owns loop quality: exploration, companion behavior, quest pacing, mini-games.
- Systems Engineer (`Codex`)
  - Owns save/load, state consistency, progression, command systems.
- World Designer (`Codex`)
  - Owns level readability, route flow, landmark density, spatial objectives.
- QA + Review Cell (`Codex + subagents`)
  - Owns bug finding, regression checks, and refinement cycles before declaring done.

## Working Protocol

1. Functionality first, then presentation polish.
2. Every feature ships with fail-safe behavior.
3. Every loop gets clear player feedback.
4. Every major pass includes a review and patch cycle.
5. Progress is only considered real when replay/reset/load all still work.

## Current Sprint Target

- Complete a stable vertical slice with:
  - companion behavior
  - scavenging + chase systems
  - objective/win state
  - persistent progression
  - reliable replay/reset
