# What Makes A Good Game

For this project, a good game is not defined by graphics first. It is defined by repeatable fun, identity, and reliable systems.

## Pillars

1. Clear fantasy
   - The player should instantly understand who they are and why the world is fun.
   - In this game: you are Kai and this neighborhood is personal.

2. Responsive feel
   - Movement, input timing, and feedback must feel reliable.
   - If controls feel bad, no amount of content fixes it.

3. Strong core loop
   - Explore -> discover -> interact -> reward -> new goal.
   - The loop must stay readable and rewarding across replay.

4. Readable goals
   - The player always knows what to do next.
   - Objective text, markers, prompts, and world cues must agree.

5. Consequences and progression
   - Actions should change state and unlock new behavior.
   - Save/load should preserve this without corruption.

6. Personality
   - Character identity and relationships matter.
   - Kai, Yuki, and Saiya should feel distinct in behavior and dialogue.

7. Replay reliability
   - Reset, load, and restart should never brick progress or break content.
   - A game is not finished if replay loops fail.

8. Scope discipline
   - Ship finished slices, not endless half-features.
   - Complete systems before adding speculative new ones.

## Definition Of Done For A Feature

- It works in normal play.
- It works after reset.
- It works after save/load.
- It gives clear player feedback.
- It does not break existing systems.
